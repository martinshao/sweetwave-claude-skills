# P5: UI

按模块调度计划执行。

1. 要求模块 Design 为 READY。
2. 将当前节点写为 `P5_UI`，UI 状态写为 `DRAFT`。
3. 调用 `/sw-ui-design-engineer` 生成或更新 `UI.md`，必要时生成 prototype。
4. 覆盖页面、组件、布局、状态、响应式、可访问性和外部设计稿引用。
5. 对 `app-shell` 定义根布局、顶部栏、侧边栏、面包屑、内容区、404、
   必要错误边界和占位页面规范；不得设计具体业务功能。
6. 不编造品牌规范；没有依据时明确标为假设。
7. 校验 DESIGN 与 UI 的页面、流程、状态一致性后写为 READY。
8. 记录 UI 指纹和下游架构影响。

`--stage ui` 完成目标范围后停止。
