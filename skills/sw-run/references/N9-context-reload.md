# N9: Context Reload

在 `RUN_STATE.md` 写入最小上下文恢复摘要：

```md
## 上下文恢复摘要

- 当前模块：
- 刚完成任务：
- 下一任务：
- 范围模式：
- 目标模块：
- 目标任务：
- 必读规格：
- 相关 LESSONS：
- Git 基线：
- 未解决风险：
- 推荐恢复命令：
```

后续任务只加载摘要指定的模块规格、任务正文、相关 LESSONS 和必要代码。
环境支持时可在状态落盘后清理会话上下文；不支持时继续执行，但不得依赖旧聊天内容。

将当前节点写为 `N9_CONTEXT_RELOAD`，然后按范围模式决策：

1. `NEXT_PROJECT_TASK`、`NEXT_MODULE_TASK`、`SINGLE_TASK`：
   - 将项目阶段恢复为 `READY_TO_RUN`。
   - 将 `RUN_STATE.md` 重置为 `IDLE`，保留完成摘要、原范围和最终 Git 基线。
   - 输出下一条建议命令并停止，不得继续 N2 或进入 N10。
2. `ALL_MODULE_TASKS`：
   - 目标模块仍有可执行任务：立即返回 N2，不向用户输出刚完成任务的执行结果或总结。
   - 目标模块无未完成活动任务：确认最后任务已按 N7 生成必要的模块完成 QA 证据，
     更新模块完成状态，
     将项目阶段恢复为 `READY_TO_RUN`，重置 `RUN_STATE.md` 为 `IDLE` 并停止。
   - 其他模块即使有任务也不得继续调度。
3. `ALL_PROJECT_TASKS`：
   - 全项目仍有可执行任务：立即返回 N2，不向用户输出中间任务结果。
   - 全项目任务已完成：进入 N10。

范围内只剩 `[BLOCKED]` 或存在未满足依赖时不得视为完成；保留阻塞检查点并停止。

## 最终输出门禁

在 `ALL_MODULE_TASKS` 或 `ALL_PROJECT_TASKS` 中，只有以下情况允许结束当前调用：

- 当前范围全部完成；
- 任务或质量门进入 `[BLOCKED]`；
- 出现必须由用户决定的业务歧义、破坏性操作或范围扩大；
- 工具或环境错误导致无法继续。

Engineer Skill 的 `COMPLETED`、任务进入 `[VERIFYING]`、单任务写入 `[x]`、
生成上下文恢复摘要，都不允许结束批量调用。
